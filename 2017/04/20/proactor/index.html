proactor笔记

发表于 2017-04-20

Proactor

An object behavioral pattern for demultiplexing and dispatching handlers for Asynchronous Events

develope high-performance web servers requires the resolution of the following forces:
  • Concurrency
  • Efficiency
  • Programming simplicity
  • Adaptablity

Concurrency strtegies :

  1. Multiple synchronouns threads
  2. reactive synchronous event dispatching
  3. proactive asynchronous event dispatching

Drawbacks

  1. Multiple Synchronous Threads
    • Operation will blocks until it completes
    • simplification of application code
    • Threading policy is tightly coupled to the concurrency policy
    • Increased synchronization complexity
    • Increased performance overhead - context switching ,synchronization ,and data movement among CPUS
    • Non-portability
  2. Reactive Synchronous Event Dispatching
Steps
  1. Registers an Acceptor with Initiation Dispathcer
  2. Web Server invokes Event Loop of the Initiation Dispatcher
  3. Acllient connect to the Web Server
  4. The Acceptor is notified by the Initiation Dispatcher of the new connection request and the Acceptor accpts the new connection
  5. The Acceptor creates an Http Handler to service the new client
  6. Http Handler register the connection with the Initiation Dispatcher for reading client request data
  7. The Http Handler service the request from the new client

for reading :

  1. The client sends an Http Get request
  2. The Initiation Dispatcher notifies the Http Handler when client request data arrives at the server
  3. The request is read in a non-blocking manner such that the read operation returns EWOUBLOCK if the operation woulod case the calling thread to block
  4. The Http Handler the Http request
  5. The requested file is synchronoously read from the file system
  6. The Http Handler registers the connection with the Initiation Dispatcher for sending file data
  7. The Initiation Dispatcher notifies the Http Handler when the Tcp connection is read for writing
  8. The Http Handler sends the request file to the client client in a non-blocing manner such that the write operation returns EWOULDBLOCK if the operation would cause the calling thread to block
    drawbacks
  9. complexing proramming
  10. lack of OS support for multi-threading
  11. scheduling of runnable tasks - reactive 是单线程，所以reactive 需要负责任务的调度，避免长时间的阻塞。　　

当ＯＳ支持异步操作时，proactor 模式会比较高效。

Poractor for web Server
  1. the web sever instructs the Acceptor to initiate an asynchronous accept
  2. the Acceptor initiates an asynchronous accept with the OS and passes itself as a Completion Handler and a reference to the Completion Dispatcher that will be used to notify the Acceptor upon completion of the asynchronous accept
  3. WS invokes the event loop of the Completion Dispatcher
  4. client connect to the WS
  5. when the asynchronous accept the operation completes ,the OS notifies the Completion Dispatcher
  6. the Completion Dispatcher notifies the Acceptor
  7. the Acceptor create an Http Handler
  8. the Http Handler initiates an asynchronous operation to read the request data from the client and passes itself as a Completion Handler and a reference to the Completion Dispatcher that will be used to notify the Http Handler upon completion of the asynchronous read

for read request:

  1. client send GET request to WS
  2. after the reading completing ,the OS notified the Completed Dispatcher
  3. Completion Dispatcher notifies the Http Handler
  4. Http Handler parse the request
  5. Http Handler synchronously read the requested file
  6. Http Handler initiates an asyncrhonous operation to write the file data to the client connection and passes itself as a Completion Handler and a reference to the Completion Dispatcher that will be used to notify the Http Handler upon completion of the asynchronous write
  7. when write operation completes ,the OS nofifies the Completion Dispatcher
  8. the Completion Dispatcher notify the Http HAndler
