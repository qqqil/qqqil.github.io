public class RodCutting {
    private int rodCutting(int[] p,int n){
        if(n == 0) return 0;
        int q  = Integer.MIN_VALUE;

        for(int i=0;i<=n;i++){
            q = Math.max(q,p[i]+rodCutting(p,n-i-1));
        }
        return q;
    }
    private int memRodCutting(int[] p,int n){
        int[] r = new int[p.length];
        for(int i =0;i<p.length;i++){
            r[i] =Integer.MIN_VALUE;
        }
        return memRodCuttingAux(p,n,r);
    }
    private int memRodCuttingAux(int[] p,int n,int[] r){
        if(r[n] >0) return r[n];
        int q = Integer.MIN_VALUE;
        if(n == 0) return 0;
        else {
            for(int i=0;i<n;i++){
                q = Math.max(q,p[i]+memRodCuttingAux(p,n-i-1,r));
            }
        }
        r[n] = q;
        return q;
    }

    private int botomUpRodCut(int[] p,int n){
        if(n ==0) return 0;
        int[] r = new int[n+1];
//        for(int i=0;i<n+1;i++){

//        }

        for(int i=1;i<=n;i++){
            int q = Integer.MIN_VALUE;
            for(int j=0;j<i;j++){
                q = Math.max(q,p[j]+r[i-j-1]);
            }
            r[i] = q;
        }
        return r[n];
    }
}

索引更有效

发表于 2017-03-20

spring 事务管理

数据库事务是指一系列的操作被当做一个整体单元执行，这些执行单元要么全部执行，要么全部失败。
ACID:

  1. Atomicity - 原子性,执行单元要么失败，要么成功；
  2. Consistency- 一致性；
  3. Isolation - 隔离性；

  4. Durability - 持久性，当事务提交完成时，事务的数据应该持久化，不会因为系统的失败而丢失。

    事务的隔离性：

    1. TransactionDefinition.ISOLATION_DEFAULT

    2. TransactionDefinition.ISOLATION_READ_COMMITTED
      避免脏读

    3. TransactionDefinition.ISOLATION_READ_UNCOMMITTED
      可能发生脏读，幻读和重复读
    4. TransactionDefinition.ISOLATION_REPEATABLE_READ
      避免可重复读，无法避免幻读
    5. TransactionDefinition.ISOLATION_SERIALIZABLE
      可避免幻读

      隔离解释：

      脏读： 读到未提交数据
      不可重复读： 在一个事务内能够读到新commit的数据，读到不同的数据
      幻读： 在一个事务内，多次读到不同的数据
